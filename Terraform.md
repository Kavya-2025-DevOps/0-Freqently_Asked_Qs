##### Frequently Asked Q&A 

1. What is Terraform?
Terraform is an Infrastructure as Code (IaC) tool by HashiCorp used to provision and manage infrastructure using declarative configuration files.

2. What is Infrastructure as Code?
Managing infrastructure using code instead of manual processes


3. What language does Terraform use?
HCL (HashiCorp Configuration Language)

4. What are providers in Terraform?
Providers are plugins that interact with APIs (AWS, Azure, GCP, Kubernetes).

5. What is remote backend?
Stores state remotely (S3, Terraform Cloud).

6. What is a resource? 
A resource is a component of infrastructure (e.g., EC2 instance, S3 bucket).


7. What is a Terraform state file? 
It stores the mapping between configuration and real infrastructure.


8. What is terraform init? 
Initializes the working directory and downloads providers.


9. What is terraform plan? 
Shows execution plan before applying changes.


10. What is terraform apply? 
Executes the changes to create/update infrastructure.


11. What is terraform destroy? 
Deletes all resources managed by Terraform.


12. What is a variable in Terraform? 
Used to parameterize configurations.


13. What is output? 
Displays useful values after execution.

14. What is a module? 
Reusable Terraform code block.

15. Difference between module and resource? 
"Resource → single infra component
Module → collection of resources"

16. What is terraform validate? 
Checks syntax and configuration correctness.
